

# Guidelines  

- Put these in a dedicated `Snippets/` or `Library/` folder.  
- Include metadata in YAML—then use Dataview (or tags) to query/search across languages or use cases :contentReference[oaicite:12]{index=12}.
- Optionally embed these snippets in permanent notes via `![[Snippet‑title]]` if you want context-rich explanation around them.

---

## 🧰 How to install these templates

1. **Save** each snippet in your vault’s Templates folder, e.g.:
   - `templates/fleeting.md`
   - `templates/zettel_permanent.md`
   - `templates/snippet_code.md`

2. **Use Templater or QuickAdd** to bind:
   - e.g. a /quick-capture shortcut to create a new Fleeting Note.
   - A “New Zettel” command to clone `zettel_permanent` into `/Permanent/` and prompt for a title.

3. Set **default new‑note location** to your Zettelkasten or Inbox folders in Obsidian settings.

---


## ✅ How these play into your Python learning journey

- **While coding or learning**: capture quick hurdles or aha‑moments with the *Fleeting template*.
- **In your end-of-day review**: convert promising notes into Zettelkasten *permanent notes* with code examples, explanations, and links (e.g. “Python‑OOP‑hashing nuance”, “Bug‑X logger output in password manager”, “Random‑list comprehension trick”).
- **When you build a small project**: document reusable code via *Snippet notes*, then connect to the concepts in your Zettel space.
- **When you revisit older notes**: enrich them progressively—split long notes into sub-notes, link them in but keep each atomic as you deepen your understanding.
